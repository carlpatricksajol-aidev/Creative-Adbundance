Static Ads Generator

Full Execution Playbook — start to finish

Internal operator runbook · Creative Adbundance

This is the complete process for producing on-brand static ad creatives for a client, from the moment they sign on to delivering finished ads. It is written from the operator's seat: what you do, in what order, and the hard-won rules that make it actually work.

1. What the system does

A client submits a short intake form. An automation (n8n) pulls that client's brand profile and a set of proven ad templates, writes on-brand ad copy, and renders finished ad images with an AI image model (KIE AI). The output is a batch of ready-to-review static ads that match the client's brand colors, fonts, voice, and winning formats. You review, pick the keepers, and deliver.

4. Phase 1 — Onboarding and intake

  1. Run the kickoff / creative brief. Capture goal, channels, audience, offer, product lines, competitors, dos and donts, and any hard rules (for example, "real talent only, no AI avatars").
  2. Collect assets: logo, brand guidelines if any, top-performing past ads ("winning ads"), and product images for product brands.
  3. The intake form sends: client name, brief, generation mode, any uploaded references, product images, and an optional UGC avatar.

5. Phase 2 — Build the Brand Brain (most important step)

The quality of every ad is decided here. One Airtable row per client, filled from the brief plus live research off their website.

Fields to fill

Auto-fill brand colors and fonts from the website

Do not eyeball colors off a logo. Scrape the live site, it carries the full real palette and typefaces and never expires. The scraper fetches the homepage plus its CSS, ranks the real brand colors (dropping neutral greys, white, and black), and pulls the fonts from Google Fonts links, @font-face, and the most-used font-family. It writes primary / secondary / accent_color_hex and brand_fonts straight into Airtable.

Why this beats the logo: a logo is often one or two colors and lives on Airtable (which expires). The website gives primary, supporting, and accent colors plus the actual fonts, from a URL that stays alive.

6. Phase 3 — Choose the generation mode

ModeUsesBehavior
top_performers (default)Proven CreativeOS template libraryLoose blueprint. 3 copy angles each (pain / benefit / trust). Best when the client has no strong references of their own.
team_top5The client's own winning_adsClose reproduction of their best ads, swapped onto current brand. 1 variant each, uses all winners.
user_referenceReferences the client uploaded this runClose reproduction. Keep the concept, swap only the brand/object. This is the highest-value mode.
Rule: if a reference is uploaded, it is always used (it auto-routes to close-reproduction). Keep the concept device; swap only the object and brand; match the original's plainness (do not add a logo box or CTA the original did not have).

2 Search Brand BrainPulls the client's row (matches on client name or sister brand name).

8. Phase 5 — The gotchas that make it work

These are the lessons that took the most time. Honor them and the pipeline runs clean.

Airtable image URLs expire (about 2 hours)

Templates, logos, and winning ads live on Airtable, and those links die fast. If KIE tries to fetch an expired link it fails with "Image fetch failed" or a 410. Fix: rehost every Airtable image to KIE within the same fresh run (the Rehost stage), then feed KIE its own permanent URLs.

Rehosted files need a real file extension

KIE names a rehosted file from the source URL. Airtable URLs have no extension, so the file lands with none and KIE rejects it ("File type not supported"). Fix: pass an explicit filename ending in .png when rehosting.

Reading an Airtable image with AI

Anthropic/Claude cannot fetch Airtable images (blocked by robots). Fix: use openai/gpt-4o for any vision read of an Airtable asset, or rehost to KIE first and read that.

Testing a single node uses stale data

When you run one node in isolation, it uses the last cached input, so Airtable URLs are already expired. Fix: re-run Search Brand Brain first to refresh the links, or run the whole flow from the trigger.

Creative direction by brand type

For human-service brands (health, tutoring, etc.), push photoreal editorial photography and real people. Ban flat vector icons, clip-art, and decorative blobs. Match the template's structure, not its pixels.

9. Phase 6 — Review and deliver

  1. Finished ads land in Supabase output. Review against the brief and brand.
  2. Pick the keepers; flag any that drifted off-brand (wrong colors, fabricated text, off-concept).
  3. Deliver for client review (statics via Figma in current workflow).
  4. Log what worked and what failed so the next batch improves.
